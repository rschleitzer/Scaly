Overview

Expressions are the core computational units of Scaly. Every expression evaluates to a value. Unlike many languages, Scaly uses postfix operators and does not define operator precedence â€” evaluation order is always explicit.

Literal Expressions

The simplest expressions are literals. A literal evaluates to itself.

Boolean Literals

The boolean literals are true and false. 

true            ; evaluates to true
false           ; evaluates to false

Integer Literals

Integer literals are sequences of decimal digits, optionally preceded by a minus sign for negative numbers. 

42              ; evaluates to 42
0               ; evaluates to 0
-7              ; evaluates to -7

Hexadecimal Literals

Hexadecimal literals start with 0x or 0X followed by hexadecimal digits (0-9, a-f, A-F). 

0xFF            ; evaluates to 255
0x1A            ; evaluates to 26
0x0             ; evaluates to 0

Floating-Point Literals

Floating-point literals contain a decimal point separating the integer and fractional parts. 

3.14            ; evaluates to 3.14
2.0             ; evaluates to 2
0.5             ; evaluates to 0.5

String Literals

String literals are enclosed in double quotes. They may span multiple lines, with line breaks preserved in the resulting string. 

"hello"         ; evaluates to "hello"
"Hello, World!" ; evaluates to "Hello, World!"
""              ; evaluates to "" (empty string)

Escape sequences are supported: \\ (backslash), \" (double quote), \' (single quote), \n (newline), \r (carriage return), \t (tab), and \0 (null). 

"line one
line two"       ; multiline string with embedded newline
"tab\there"     ; string with tab character
"say \"hi\""    ; string with escaped quotes

Character Literals

Character literals represent a single character, enclosed in backticks. 

`a`             ; evaluates to "a"
`Z`             ; evaluates to "Z"

Fragment Literals

Fragment literals are used within string interpolation to represent literal text segments between interpolated expressions.
