Overview

Expressions are the core computational units of Scaly. Every expression evaluates to a value. Unlike many languages, Scaly uses postfix operators and does not define operator precedence â€” evaluation order is always explicit.

Note: This chapter will be expanded as expression support is implemented. Currently, only literal expressions are fully supported.

Literal Expressions

The simplest expressions are literals. A literal evaluates to itself. 

42              ; evaluates to 42
"hello"         ; evaluates to "hello"
true            ; evaluates to true

Operator Expressions

Status: planned

Operators in Scaly are postfix: they follow their operands. This eliminates precedence ambiguity entirely. 

; Planned syntax:
3 4 +           ; addition: 7
3 4 + 2 *       ; (3 + 4) * 2 = 14

Because operators are postfix, the evaluation order is always left-to-right. No parentheses are needed to override precedence because there is no precedence.

Grouping

Status: planned

Parentheses can be used for grouping to change evaluation order when needed. 

; Planned syntax:
3 (4 2 *) +     ; 3 + (4 * 2) = 11

Function Application

Status: planned

Functions are applied prefix-style: the function name precedes its arguments. 

; Planned syntax:
sqrt 16         ; function application: 4
max 3 7         ; two-argument function: 7

The combination of prefix functions and postfix operators creates a natural flow: data flows left-to-right through transformations.
